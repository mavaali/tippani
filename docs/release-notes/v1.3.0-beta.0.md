## Tippani 1.3.0-beta.0 — WYSIWYG editing & write path (beta)

Tippani can now **read, annotate, *and* edit** ADO PR specs. This is the first release that writes to ADO branches — shipped as a **beta** so it can be exercised on real PRs before becoming the default install.

> ⚠️ **Beta.** Editing commits to the PR's source branch. Verified end-to-end against a live PR, but treat important specs with care and please report anything off.

### What's new

- **Live-preview editor** — a Typora-style CodeMirror 6 editor in the spec view. The editor's document *is* the markdown file, so saves produce clean diffs and YAML frontmatter, HTML comments, and ADO macros (`[[_TOC_]]`, `::: mermaid`, mentions) pass through untouched. Formatting renders inline and markup reveals on the cursor's line.
- **WYSIWYG tables** — pipe tables become an editable grid: tab/arrow navigation, add/delete rows and columns, column alignment — round-tripping to canonical pipe markdown. An unedited table is never reformatted.
- **Edit / view toggle** — read-only render stays the default; editing is opt-in via the header button or `Cmd`/`Ctrl`+`E`.
- **Save to the PR branch** — commit edits straight to the PR source branch, with a diff-on-save preview and an editable commit message. Explicit save only.
- **Dirty state & conflict guard** — unsaved-change indicators and prompts, plus optimistic-concurrency protection: a push made stale by someone else's commit is rejected and you're offered reload / copy-to-clipboard. Never auto-merges.
- **Offline-aware edit gating** — Edit is offered only with repo push access; offline edits queue and sync on reconnect; finished PRs aren't editable.

### Known limitations

- The per-branch push ACL isn't pre-checked — the permission probe is repo-level and fails open; a real push rejection still surfaces gracefully at save time, and the edit is never lost.
- Fenced code blocks are styled but not language-tokenized.

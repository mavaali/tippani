## Tippani 1.4.0-beta.0 — AI / MCP integration (beta)

Tippani now exposes an HTTP control API and an MCP server so LLM tools (Claude Desktop, GitHub Copilot, scripts, IDE extensions) can drive the review workflow while the user stays in tippani's browser UI and approves every action. The Phase 0 keyboard nav also lands here so single-handed comment triage works with or without an LLM in the loop.

> ⚠️ **Beta.** The MCP path is brand new. Tested end-to-end against the in-memory control API; the live Claude Desktop / Copilot wiring has not been exercised against a real PR yet. Test on non-critical PRs first.

### Design

[Issue #42](https://github.com/mavaali/tippani/issues/42) — *Control API for AI-assisted comment resolution*. Three phases, all shipped together:

- **Phase 0** — keyboard nav + inline reply textarea (zero-LLM dependency)
- **Phase 1** — HTTP control API under `/api/v1/*` with session-token auth
- **Phase 2** — `tippani-mcp` stdio server exposing 8 tools

The core principle: **tippani is the server, the LLM is the client**. tippani has no embedded LLM and never auto-posts. Drafts are staged; the user approves in the UI.

### What's new

#### Keyboard navigation (Phase 0)

| Key | Action |
|---|---|
| `J` / `S` | Next active thread (scroll + highlight) |
| `K` | Previous active thread |
| `R` | Open reply form on focused thread |
| `⌘`/`Ctrl` + `Enter` | Post reply and auto-advance |
| `Esc` | Cancel reply |

The old `prompt()` dialog is gone — replies happen inline. Focused-thread state survives the post-reply page reload so "Post & next" lands on the right thread.

#### HTTP control API (Phase 1)

All routes under `/api/v1/*`:

| Method | Path | Purpose |
|---|---|---|
| `GET` | `/threads` | List threads (summary + current focus) |
| `GET` | `/threads/:id` | Full thread (comments + any staged draft) |
| `PUT` | `/threads/:id/draft` | Stage a reply draft (409 if user editing) |
| `DELETE` | `/threads/:id/draft` | Clear staged draft |
| `POST` | `/threads/:id/lock` | Touch user-editing lock (10s sliding TTL) |
| `POST` | `/threads/:id/reply` | Post reply to ADO (409 if in-flight) |
| `POST` | `/threads/:id/resolve` | Resolve thread |
| `POST` | `/commands/focus` | Scroll browser to thread |
| `GET` | `/specs/:fileIndex` | Structured JSON (markdown + headings) |
| `GET` | `/state` | Browser polls this every 1.5s for focus + drafts |

**Auth.** A random 24-byte session token is generated at startup and written to `~/.tippani/session-token` (mode 0600, cleaned up on graceful shutdown). External clients send `Authorization: Bearer <token>` plus `X-Tippani-Client: <name>`. The browser uses same-origin and needs neither.

**Externally-staged drafts in the UI.** When an external client stages a draft, the browser picks it up via the 1.5s poll, populates the reply textarea, and shows a "✨ Draft from external client" badge. The user always edits or posts.

#### MCP server (Phase 2)

`tippani-mcp` is a stdio MCP server exposing 8 tools:

`list_threads`, `get_thread`, `focus_thread`, `stage_draft`, `clear_draft`, `post_reply`, `resolve_thread`, `get_spec`

One-line setup in Claude Desktop's `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "tippani": { "command": "npx", "args": ["-y", "tippani-mcp"] }
  }
}
```

Start tippani first; the shim reads the token from `~/.tippani/session-token`. No copy/paste.

### Try it

```bash
npm i -g tippani@next
tippani <PR_ID>
# then in Claude Desktop / Copilot, ask: "list the open threads on the PR I have open in tippani"
```

### Known limitations

- The browser uses 1.5s polling for focus + draft state. SSE / long-poll is intentionally deferred — MCP clients handle real-time fine via tool calls.
- `post_reply` and `resolve_thread` don't roundtrip through a confirmation in the UI — they post directly. Prefer `stage_draft` for the "user approves in the UI" flow.
- No MCP transport smoke test in CI yet — the tool surface is covered by `src/mcp.test.mjs` against the real HTTP API, but the stdio MCP wire format itself isn't exercised end-to-end. First Claude Desktop session is the smoke test.
